<ol>
<li>
<span data-as="p">Enter an <strong>email address</strong>: Choose an email address for the user who will

<span data-as="p">Click <strong>Send email OTP</strong>: The Web SDK generates a unique <code>deviceId</code>, which

verification tokens (<code>deviceToken</code>, <code>deviceEncryptionKey</code>, <code>otpToken</code>)
used by the Web SDK to verify the OTP.</span>
</li>
<li>
<span data-as="p">Open your <strong>Mailtrap Sandbox → Inbox</strong> to view the OTP email sent by

unless email forwarding is configured.</span>

OTP codes directly to users’ email inboxes.</span></div></div>
</li>
<li>
<span data-as="p">Click <strong>Verify email OTP</strong>: Circle opens a hosted OTP verification window.
The user enters the code they received. The Web SDK verifies the OTP with
Circle, which then returns a <code>userToken</code> and <code>encryptionKey</code>. Together,
they enable an authenticated Circle user session.</span>
</li>
<li>
<span data-as="p">Click <strong>Initialize user</strong>: Your backend initializes the user using the

loads the existing wallet instead.</span>
</li>
<li>
<span data-as="p">Click <strong>Create wallet</strong>: The Web SDK executes the challenge using the
<code>challengeId</code>. The user approves the action, and Circle creates the
